# Deployment Skill

## Render.com Deployment (Production)

### Initial Setup

1. **Connect GitHub Repository**
   - Go to Render Dashboard
   - New → Web Service
   - Connect GitHub repo: `Shopify_connector-main`
   - Branch: `main`

2. **Configure Service**
   - Name: `shopify-connector`
   - Runtime: Docker
   - Region: Choose closest to users
   - Instance Type: Starter (upgrade as needed)

3. **Set Environment Variables**
   Required variables in Render dashboard:
   ```
   FLASK_ENV=production
   FLASK_SECRET_KEY=<generate-random-key>
   MERGADO_CLIENT_ID=<from-mergado-dev-portal>
   MERGADO_CLIENT_SECRET=<from-mergado-dev-portal>
   MERGADO_REDIRECT_URI=https://<your-service-name>.onrender.com/auth/callback
   # (Must match Mergado app settings; use /auth/callback path.)
   DATABASE_URL=<auto-generated-by-render-postgres>
   LOG_LEVEL=INFO
   LOG_FORMAT=json
   ```

4. **Add PostgreSQL Database**
   - In Render dashboard: New → PostgreSQL
   - Link to web service
   - DATABASE_URL is auto-populated

5. **Deploy**
   - Render auto-deploys on push to `main`
   - Manual deploy: Dashboard → Manual Deploy

### Deployment Checklist

Before deploying to production:

- [ ] All tests passing: `pytest --cov=app --cov-fail-under=80`
- [ ] Code formatted: `black . && isort .`
- [ ] Linter clean: `flake8 app/`
- [ ] Database migrations ready: Check `migrations/versions/`
- [ ] Environment variables set in Render
- [ ] Mergado OAuth redirect URI updated
- [ ] Health check endpoint working: `/health`
- [ ] Logs configured (JSON format for production)

### Running Migrations on Render

Render doesn't auto-run migrations. Two options:

**Option 1: Run via Render Shell**
```bash
# In Render dashboard → Shell
flask db upgrade
```

**Option 2: Add to Dockerfile (recommended)**
```dockerfile
# In Dockerfile, before CMD
RUN flask db upgrade
```

### Monitoring

**Health Check**
- Endpoint: `GET /health`
- Render checks this every 30s
- Returns: `{"status": "healthy", "service": "mergado-shopify-connector"}`

**Logs**
- View in Render Dashboard → Logs
- JSON format in production for better parsing
- Key fields: `timestamp`, `level`, `message`, `request_id`

**Metrics**
- Render provides: CPU, Memory, Response Time
- Add custom metrics via Mergado notifications API

### Rollback

If deployment fails:

1. **Quick Rollback**
   ```bash
   git revert <bad-commit-hash>
   git push origin main  # Triggers redeploy
   ```

2. **Render Dashboard Rollback**
   - Go to Deploys
   - Find last working deploy
   - Click "Redeploy"

3. **Database Rollback**
   ```bash
   # Via Render Shell
   flask db downgrade  # Go back one migration
   ```

### Debugging Deployment Issues

**Container won't start:**
- Check Render logs for errors
- Verify Dockerfile builds locally: `docker build -t test .`
- Check health endpoint is accessible

**Database connection errors:**
- Verify DATABASE_URL is set
- Check PostgreSQL service is running
- Test connection from Render Shell: `flask db current`

**OAuth failures:**
- Verify MERGADO_REDIRECT_URI matches Render URL
- Check client ID/secret are correct
- Ensure OAuth scopes are configured in Mergado Dev Portal

## Local Docker Deployment (Testing)

### Build and Run
```bash
# Build image
docker build -t shopify-connector .

# Run container
docker run -p 5000:5000 --env-file .env shopify-connector

# Check health
curl http://localhost:5000/health
```

### Docker Compose (with PostgreSQL)
```yaml
# docker-compose.yml
version: '3.8'
services:
  web:
    build: .
    ports:
      - "5000:5000"
    environment:
      - DATABASE_URL=postgresql://user:pass@db:5432/shopify_connector
    depends_on:
      - db
  
  db:
    image: postgres:15
    environment:
      - POSTGRES_USER=user
      - POSTGRES_PASSWORD=pass
      - POSTGRES_DB=shopify_connector
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```

Run: `docker-compose up`

## CI/CD Pipeline

GitHub Actions (`.github/workflows/ci.yml`) runs on push:

1. **Test Job**
   - Install dependencies
   - Run pytest with coverage
   - Upload coverage report

2. **Docker Job** (depends on Test)
   - Build Docker image
   - Start container
   - Health check test

3. **Deploy** (only on main branch)
   - Render auto-deploys if tests pass

## Environment Management

### Development (.env file)
```env
FLASK_ENV=development
FLASK_DEBUG=true
DATABASE_URL=sqlite:///dev.db
MERGADO_REDIRECT_URI=http://localhost:5000/callback
```

### Production (Render)
- Set via Render Dashboard → Environment
- Never commit production secrets to git
- Use Render's secret management

## Database Backups

**Render PostgreSQL**
- Automatic daily backups (retained 7 days on free tier)
- Manual backup via Render dashboard
- Restore from backup: Dashboard → Backups → Restore

**Manual Backup**
```bash
# Via Render Shell
pg_dump $DATABASE_URL > backup.sql

# Restore
psql $DATABASE_URL < backup.sql
```

## Scaling

**Vertical Scaling (More Power)**
- Render Dashboard → Instance Type
- Upgrade from Starter → Standard → Pro

**Horizontal Scaling (More Instances)**
- Add workers for background jobs (Celery)
- Load balancer (Render handles this automatically)

**Database Scaling**
- Upgrade PostgreSQL plan in Render
- Add read replicas for heavy read workloads

## Security Checklist

Before production:
- [ ] FLASK_SECRET_KEY is strong random value (not default)
- [ ] All OAuth secrets in environment (not code)
- [ ] HTTPS only (Render provides this)
- [ ] CSRF protection enabled
- [ ] SQL injection protected (using SQLAlchemy ORM)
- [ ] Rate limiting on API endpoints
- [ ] Input validation on all forms
- [ ] Logs don't contain tokens or secrets

## Common Issues

**"Application Error" in Render**
- Check logs for stack trace
- Verify all env vars are set
- Test locally with same env vars

**Database migrations not applied**
- Run manually: `flask db upgrade` in Shell
- Or add to Dockerfile

**OAuth redirect mismatch**
- Update redirect URI in Mergado Dev Portal
- Ensure matches Render URL exactly (including https://)

**Slow performance**
- Check Render metrics (CPU/Memory)
- Optimize database queries (add indexes)
- Enable caching
- Consider upgrading instance
